# NexusRealtime Ideas Workflow

## Core loop

```text
read repo brain
choose one scope folder
walk related domains
record catalogue findings
create or update candidate packet
update saved state
stop
```

## Scope rule

Every meaningful item must be placed in a current-scope folder.

Use this cascade:

```text
00-inbox -> 10-atomic-domains -> 20-domain-families -> 30-composite-loops -> 40-host-bridges -> 50-render-descriptors -> 60-incubation-suites -> 70-promotion-candidates -> 80-blocked-or-held
```

## Catalogue-first rule

Default mode is catalogue-only.

Builder mode is allowed only when a saved state explicitly says `Builder status: ready`.

## No-go rules

- Do not create scheduled tasks from this repo unless explicitly instructed.
- Do not edit NexusRealtime Core, ProtoKits, or Experiments.
- Do not claim promotion readiness without a candidate packet and proof notes.
- Do not store scope only in prose. Use folders.
